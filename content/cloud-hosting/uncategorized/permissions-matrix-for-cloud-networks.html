The following permissions matrix displays specific permissions for the roles in Cloud Networks. The matrix displays the method names, their corresponding RESTful API commands, and the roles that are supported.

The following permissions matrix displays specific permissions for roles using the Neutron API.

NETWORKS
List Networks
GET /networks
 Lists networks to which the specified tenant has access.
List Details for a Network GET /networks/{network-id} Shows information for a specified network ID.
Create a Network POST /networks   Creates a network.
Update a Specified Network PUT /networks/{network-id}     Updates editable attributes for a specified network.
Delete a Specified Network DELETE /networks/{network-id}     Deletes the specified network and its associated resources.

SUBNETS

List Subnets

 GET /subnets Lists subnets to which the specified tenant has access.
List Details for a Subnet GET /subnets/{subnet-id} Shows information for a specified subnet.
Create a Subnet POST /subnets   Creates a subnet on a specified network.
Update a Specified Subnet PUT /subnets/{subnet-id}     Updates editable attributes for a specified subnet.
Delete a Subnet DELETES /subnets/{subnet-id}     Deletes a specified subnet.

PORTS
List Ports GET /ports Lists ports to which the tenant has access.
List Details for a Port GET /ports/{port-id} Shows information for a specified port.
Create a Port POST /ports   Creates a port on a specified network.

Update Editable Attribute of a Port

 PUT /ports/{port-id}     Updates a editable attributes for a specified port.
Delete Specified Port DELETE /ports/{port-id}     Deletes a specified port.

Cloud Networks Terminology

Network

A sequence of connection points that communicate with each other.

Server

A virtual machine (VM) instance in the Cloud Servers environment. To create a server, you must specify a name, flavor reference, and image reference.

Virtual Interface

An extension to the networking API that is specifically used for attaching and detaching networks.
